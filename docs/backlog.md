# 📋 Backlog do Projeto — Elevador Inteligente Acessível (EIA)

O backlog do projeto foi organizado com base na metodologia MoSCoW, priorizando funcionalidades essenciais para o funcionamento do MVP e evolução futura do sistema.

---

## 🔴 Alta Prioridade (MVP — Essencial)

### US01 — Ativação do modo acessível
Permitir que o usuário ative o modo acessível por meio de botão físico ou sensor.

### US02 — Ajuste do tempo de abertura da porta
O sistema deve aumentar automaticamente o tempo de abertura da porta quando o modo acessível estiver ativo.

### US03 — Comunicação via MQTT
O ESP32 deve enviar eventos em tempo real para o backend utilizando protocolo MQTT.

---

## 🟠 Média Prioridade (Funcionalidades de suporte)

### US04 — Registro de eventos
O sistema deve armazenar logs de eventos (ativação, uso, estado) no backend.

### US05 — Exibição de status no aplicativo
O aplicativo deve exibir o estado atual do elevador (padrão ou modo acessível).

---

## 🟡 Baixa Prioridade (Evolução futura)

### US06 — Notificações em tempo real
Implementação de alertas para eventos críticos do sistema.

### US07 — Dashboard de monitoramento
Painel para visualização de métricas de uso e histórico de eventos.

---

## 📌 Observação
Este backlog representa a organização das funcionalidades do sistema em níveis de prioridade, permitindo evolução incremental do protótipo IoT + Mobile.
